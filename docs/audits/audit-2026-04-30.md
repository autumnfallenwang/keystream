## Recommendations (priority order)

### 1. `src/components/text-panel.tsx` → CodeMirror 6

**Signals firing (5/5):**

- **Bug-ratio inversion:** B-02 (active-line stuck on line 1) and B-03 (gutter line-height misalignment) both originated here. D-03 (gutter in edit mode) and D-13 (wrap toggle) both required structural rework. Multiple bug fixes during a single feature implementation: wrap mode broke last-line-cutoff, then required `scrollbar-gutter: stable`, then required a measure-mirror, then needed an overscroll fix. Five Q-anchored defensive comment blocks in 394 lines.

- **Edge-case avalanche:** wrap-with-line-numbers alignment forced the `useLineHeights` measurement hook + invisible mirror DOM. The auto-indent issue (Python in VSCode) is the next class-of-problem the panel can't address from inside its own boundary — that's already escalated to a documentation-only workaround in README. Each fix has uncovered the next one; the class of problems is "things off-the-shelf code editors solved a decade ago."

- **Foundation-becomes-ceiling:** syntax highlighting for Python / Rust / JSON would be a multi-week project from where we are now. Find/replace, undo history, multi-cursor, virtualization for long files — same. The `<textarea>` + `<pre>` shape blocks all of these. CodeMirror gives them as flags or one-line extensions.

- **Knowledge isolation:** the `useLineHeights` ResizeObserver / mirror-mirror geometry is documented only in inline comments. A new contributor would need to derive the design from scratch. No external docs / Stack Overflow surface — it's our private invention.

- **Defensive justification:** SKILL.md's own opening paragraph cites this file as the canonical "should have bought it" example. Inline comments include phrases like "Q21 — when wrap is on, a source line can render across multiple visual rows; the gutter…" — that's defensive geometry.

**Proposed swap:** CodeMirror 6 (`codemirror` ^6.x + `@codemirror/state` + `@codemirror/view` + `@codemirror/language` + `@codemirror/commands` + 4–5 `@codemirror/lang-*` for our text allowlist). Optional convenience wrapper `@uiw/react-codemirror` saves ~30 lines of mount boilerplate.

**Bundle cost:** ~80–120 KB gzipped (core ~50 KB + 5–15 KB per language × 5 languages). Net reduction from current panel size after deleting `useLineHeights`, the mirror, and the `LockedView`/`EditView` split.

**Effort estimate:** ~3 days (one phase). Behind the same `<TextPanel>` boundary, so `page.tsx` and the IPC layer don't change.

**Domain extensions to port:**
- Active-line scanline → `Decoration.line({ class })` driven by a `StateField` listening to reducer's `charsTyped`.
- Wrap toggle → `Compartment.reconfigure(EditorView.lineWrapping)`.
- Read-only / lock mode → `EditorState.readOnly.of(true)` (replaces our `EditView`/`LockedView` split).
- Language autodetection → extend `classifyIcon` in `src/lib/core/file-tree.ts` with `pickLanguage(name)`.
- Hidden upside: empty-state ("Drop a file here, or click to load") stays hand-rolled — CodeMirror only mounts when there's text.

**Q-decisions affected:**
- Q14 (SendControl tri-state): unchanged contract; only the visual binding moves to a CodeMirror decoration.
- Q16 (gutter shared across edit/lock + soft-wrap disabled): superseded *implementation* (CodeMirror handles the geometry natively), contract preserved (gutter visible in both modes; wrap default still off via the toggle).
- Q21 (wrap toggle, header consolidation): unchanged behavior; toggle flips the `Compartment` instead of toggling our wrap branches.

**Why now:** the text panel's bug log is the densest in the codebase, the auto-indent finding showed us the *next* class of problem we can't solve in-house, and the swap is still cheap (single-maintainer codebase, no extension API external users depend on). Post-v0.1.3 is the right window — the release pipeline is stable, no in-flight features depend on the current panel's geometry.
