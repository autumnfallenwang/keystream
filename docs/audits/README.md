# Component audits

Output of the `audit-components` skill. Each file is dated (`audit-YYYY-MM-DD.md`) and represents a point-in-time review of which hand-rolled components are candidates for off-the-shelf swaps, scored against the five-signal framework.

Run the skill via `/audit` or by saying things like "audit components" or "should we replace X". Output saved here for drift tracking — components that flip from "keep" to "swap" over time are the most interesting deltas.

See `.claude/skills/audit-components/SKILL.md` for the full framework and `references/alternatives.md` for the curated swap targets.
